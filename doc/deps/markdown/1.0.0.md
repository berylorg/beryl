# markdown 1.0.0

Verified on 2026-05-12.

## Workspace Use

- `beryl-app` uses `markdown-rs` as the Markdown parser for transcript rendering.
- The workspace depends on the crate as `markdown = { version = "1.0.0", default-features = false }`.
- Enabled `markdown` crate features: none.
- Resolved transitive dependency: `unicode-id 0.3.6`.

## Parser Entry Points

- Use `markdown::to_mdast(value, &ParseOptions)` to parse Markdown into an owned `markdown::mdast::Node`.
- `to_mdast` returns `Result<mdast::Node, markdown::message::Message>`.
- Use `markdown::ParseOptions` and `markdown::Constructs` to select parser constructs.
- Do not use `to_html`, `to_html_with_options`, `Options`, or `CompileOptions` for transcript UI rendering.

## Memory-Retention Notes

- `to_mdast` first calls `parser::parse`, which builds a temporary `Vec<Event>` and a `ParseState` borrowing the input bytes, then calls `to_mdast::compile`.
- `to_mdast::compile` allocates an owned mdast tree with `Vec<Node>` children and owned `String` values for text, code, math, HTML, links, images, labels, and titles.
- Returned mdast nodes do not borrow the source string. Positions store numeric offsets in `markdown::unist::Position`; they do not retain source slices.
- The crate does not define process-global parser caches, resource pools, `OnceLock`/`Lazy` state, or thread-local retained parse state in the source paths inspected.
- Memory during parse is proportional to input bytes, event count, nesting, and generated AST text. The crate does not provide a parse-size limit; Beryl must bound source size, cache entries, and in-flight parse copies at call sites.
- Beryl converts mdast into Beryl-owned transcript IR immediately. Beryl's `TranscriptMarkdownCache` is the long-lived retention layer, with default limits of 512 entries and 1,000,000 tracked source bytes.

## Constructs For Beryl

- `Constructs::default()` follows CommonMark and enables the core constructs Beryl needs for paragraphs, ATX and setext headings, block quotes, code spans, fenced and indented code blocks, links, images, lists, hard breaks, thematic breaks, autolinks, definitions, character escapes, and character references.
- CommonMark defaults also enable `html_flow` and `html_text`; Beryl must translate `mdast::Node::Html` into literal or unsupported-source IR and never into live UI.
- Math is off by default. Enable `math_flow` and `math_text` in Beryl parser options so block math and inline math are represented structurally from the start.
- `ParseOptions::math_text_single_dollar` defaults to `true`; Beryl sets it to `false` for transcript parsing so ordinary currency text such as `$5 and $6` remains text. Inline math can still use two or more dollar delimiters.
- MDX constructs are off by default. Keep `mdx_esm`, `mdx_expression_flow`, `mdx_expression_text`, `mdx_jsx_flow`, and `mdx_jsx_text` off for transcript Markdown.
- GFM table, task-list, footnote, autolink-literal, and strikethrough constructs are off unless `Constructs::gfm()` or individual flags are used. They are not part of the initial transcript rendering slice.
- `ParseOptions` can hold boxed MDX parse callbacks, so Beryl should build parser options locally for parsing and avoid storing callback-bearing parser options in shared UI state.

## AST Nodes Needed

- Document and blocks: `Node::Root`, `Node::Paragraph`, `Node::Heading`, `Node::Blockquote`, `Node::List`, `Node::ListItem`, `Node::Code`, `Node::Math`, `Node::ThematicBreak`, `Node::Html`.
- Inlines: `Node::Text`, `Node::Emphasis`, `Node::Strong`, `Node::InlineCode`, `Node::InlineMath`, `Node::Break`, `Node::Link`, `Node::Image`, `Node::LinkReference`, `Node::ImageReference`.
- `mdast::List` carries `ordered`, optional `start`, and `spread`.
- `mdast::ListItem` carries child blocks, `spread`, and optional GFM `checked`.
- `mdast::Code` carries `value`, optional `lang`, and optional `meta`.
- `mdast::Math` carries `value` and optional `meta`.
- `mdast::InlineMath` carries `value`.
- `mdast::Link` carries child inlines, `url`, and optional `title`.
- `mdast::Image` carries `alt`, `url`, and optional `title`.
- Reference nodes carry normalized identifiers and labels; Beryl preserves link and image references as unsupported fallback source until reference resolution is implemented.

## Integration Gotchas

- Convert `markdown-rs` AST nodes into Beryl-owned transcript IR immediately. Do not expose `markdown::mdast` types across the transcript rendering boundary.
- Positions are optional on nodes and useful for fallback source recovery, but Beryl should not require positions for basic rendering.
- `Position.offset` behaves as a Rust byte offset in the cases Beryl uses for fallback-source extraction. Keep the non-ASCII raw HTML fallback test when changing this code.
- Incomplete streaming Markdown may parse differently before the final tokens arrive. Cache keys must include source text generation or content identity, not just item id.
- The crate can compile to safe HTML by default, but Beryl must not use HTML compilation for transcript UI because the product contract requires native `gpui` rendering and no HTML rendering.
- Keep parser configuration explicit in Beryl code rather than relying on default construct behavior, especially for HTML, math, and MDX policy.
