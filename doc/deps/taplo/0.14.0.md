# Workspace Use

Beryl owns token taxonomy, themes, and rendering. This crate may supply TOML parser, syntax-tree, token, parse-error, and source-range facts only.

Use `taplo::parser::parse` as the TOML parser entry point:

- `parse(source: &str) -> taplo::parser::Parse`
- `Parse::into_syntax() -> taplo::syntax::SyntaxNode`
- `Parse::errors: Vec<taplo::parser::Error>`
- `Error::range: taplo::rowan::TextRange`
- `Error::message: String`

Use Rowan syntax traversal to collect highlight facts:

- `SyntaxNode::descendants_with_tokens()`
- `SyntaxNode::children_with_tokens()`
- `SyntaxNode::kind()`
- `SyntaxNode::text_range()`
- `SyntaxToken::kind()`
- `SyntaxToken::text()`
- `SyntaxToken::text_range()`
- `SyntaxElement::kind()`
- `SyntaxElement::text_range()`

`taplo::rowan` is re-exported, so Beryl may refer to `taplo::rowan::{NodeOrToken, TextRange, TextSize}` when converting ranges.

# Range Notes

Taplo ranges are Rowan `TextRange` and `TextSize` values. `TextSize` is a UTF-8 byte offset stored as `u32`, and Taplo builds syntax ranges from lexer spans over the original UTF-8 source.

Convert ranges to `usize` byte offsets and validate that each emitted range slices the original `&str` before constructing Beryl highlight tokens.

# Recovery And Semantics

Taplo parsing is lossless for highlighting purposes and preserves comments and whitespace in the syntax tree.

The parser collects parse errors and can emit `SyntaxKind::ERROR` tokens without stopping traversal of surrounding recoverable syntax.

`parser::parse` does not perform TOML semantic validation such as duplicate-key checks. Do not use Taplo DOM validation, formatter, analyzer, schema, or serde APIs for syntax highlighting.

# Lifecycle And Retention

`parse(&str)` returns an owned green tree plus owned parse errors. Syntax traversal is immutable.

For Beryl, parse off the `gpui` render path, immediately convert syntax tokens and errors into Beryl-owned highlight ranges, and do not store Taplo syntax nodes or green trees in long-lived code-panel cache output.

Taplo's parser source has a recursive parsing caveat for deeply nested structures. Keep TOML highlighting behind Beryl's existing code-panel source-size and cache-retention limits, and avoid treating Taplo as safe for unbounded pathological nesting.
