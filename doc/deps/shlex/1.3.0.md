# shlex 1.3.0

Verification date: 2026-05-08.

Enabled features: `default`, `std`.

## Use In This Workspace

Beryl uses `shlex::try_quote` in `beryl-backend` to encode individual strings as POSIX shell words for WSL shell command construction.

Backend tests use `shlex::split` to verify generated shell command words.

## Symbols Used

- `shlex::try_quote(&str) -> Result<Cow<'_, str>, shlex::QuoteError>` quotes one shell word with default settings.
- `shlex::split(&str) -> Option<Vec<String>>` parses a POSIX shell-like string in tests.
- `shlex::QuoteError::Nul` is returned when input contains a NUL byte.
- With `std`, `QuoteError` implements `std::error::Error`.

## Invariants And Gotchas

`try_quote` may return a borrowed `Cow` for inputs that are safe unquoted. Call `into_owned()` when a `String` is needed.

The output is POSIX-shell-safe but not fixed to a single quoting style. It can leave safe words unquoted, single-quote most metacharacter-containing words, and use double-quoted chunks for single quotes and backslashes.

NUL bytes are rejected by default. Do not use deprecated `shlex::quote`, which allows NUL bytes through.

Non-ASCII bytes are quoted. Valid UTF-8 input remains valid UTF-8, and multibyte characters are not rewritten.

Non-NUL control characters are not escaped into printable sequences. Upstream documents this as safe for noninteractive shell command strings, but unsafe for piping directly into an interactive shell.
