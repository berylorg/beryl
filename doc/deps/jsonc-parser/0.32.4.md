# JSON And JSONL Notes

Use strict parser options for the `json` label so JSON5-only constructs are not treated as valid JSON.

For `jsonl` and `ndjson`, run strict JSON highlighting per physical line and offset line-local ranges back into the original source.

# JSON5 Deferral

JSON5 support is deferred out of the active plan.

`jsonc-parser` supports comments, trailing commas, loose object property names, single-quoted strings, hexadecimal numbers, unary-plus numbers, and standard decimal/exponent numbers.

It is not a faithful full JSON5 parser. Unsupported JSON5 number forms include:

- leading decimal point numbers such as `.5`
- trailing decimal point numbers such as `5.`
- `Infinity`, `-Infinity`, and `+Infinity`
- `NaN`, `-NaN`, and `+NaN`

Unquoted keys are supported as loose words or numbers, but this is not documented as exact ECMAScript 5.1 `IdentifierName` validation.
