## Why Beryl Uses It

Beryl uses `gpui-text-input` directly in `beryl-app` for startup fields, workspace picker and rename fields, the conversation composer, and a read-only selectable surface-notice field.

Beryl also uses it indirectly through `gpui-settings-window`, whose settings fields and color-picker component inputs wrap `gpui_text_input::TextInput`.

## Retained State And Lifecycle

Each `TextInput` entity owns a `TextInputState` with the current normalized UTF-8 text buffer, selection range, selection direction, optional IME marked-text range, opaque atom vector, and undo/redo stacks.

Undo and redo retain full-buffer `EditSnapshot` values. Each snapshot clones the full text string and atom vector. `TextInputOptions::DEFAULT_UNDO_LIMIT` is `128`, and both undo and redo stacks are individually limited by that count. The limit is deterministic by snapshot count, not by retained bytes.

`TextInput::set_text` calls `TextInputState::reset_text`, which replaces the current buffer and clears undo and redo stacks. Beryl uses this on composer clear, history replacement, transcript-edit seeding, workspace picker reset, rename seeding, surface-notice sync, and settings-window value sync.

The widget also retains last-render layout data: `last_layout: Vec<InputLineLayout>`, `last_bounds`, scroll offsets, content height, and visible byte range. `InputLineLayout` owns GPUI `WrappedLine` values for the current rendered text. This is bounded by the current widget entity and current text, but there is no explicit byte or line cap in `gpui-text-input`.

Layout construction allocates transient full-text and per-line structures: it clones `state.text()` to a `String`, creates logical-line ranges, atom ranges, text runs, shaped lines, selection quads, and visual-line helper vectors. These are mostly dropped after prepaint/paint except for `last_layout`.

The crate does not define global text layout caches, maps, or resource pools. It calls GPUI `window.text_system().shape_text`; any GPUI-owned glyph or shaping cache is outside this note.

## Clipboard, Atoms, And IME

Plain copy/cut writes only `selection.copy_text()` to the GPUI clipboard unless atom clipboard policy is `Propagate` and the selection contains atoms.

Plain paste reads clipboard text and inserts it unless rich paste policy is `Propagate` and the clipboard item has non-string entries or metadata-only content is allowed. Beryl sets `Propagate` for the composer so Beryl-owned image and rich clipboard handling runs outside the dependency.

IME composition is retained only as the current `marked_range` plus current text. `replace_and_mark_text_in_range` edits the buffer and marks the inserted range; `unmark_text` clears the range. There is no separate composition string cache beyond the normal buffer and undo snapshot path.

## Integration Gotchas

Large composer or settings-field contents can be retained multiple times: current `TextInputState.text`, Beryl or settings-window mirror state, `TextInputState` undo/redo snapshots, and `last_layout` shaped lines.

The highest-impact dependency-owned retention is the default undo/redo history on multiline composer and settings text fields. Snapshot count is bounded, but a large draft edited repeatedly can retain many full historical buffers until `set_text`, entity teardown, or enough edits evict older snapshots.

Beryl can configure this at construction with `TextInputOptions::with_undo_limit`. Beryl currently sets undo limit `0` only for the read-only surface-notice input. Other direct Beryl inputs and `gpui-settings-window` inputs use the default `128`.

Opaque atoms are dependency-owned only as ids, ranges, and fallback copy strings. Image bytes, durable asset ids, label scopes, and rich clipboard payloads are Beryl-owned; Beryl's `ComposerClipboardStore` has its own bounded payload count of `32`.

## Minimal Source Entrypoints

- `gpui-text-input/src/options.rs`: `TextInputOptions`, default undo limit, configuration methods.
- `gpui-text-input/src/state.rs`: `TextInputState`, `EditSnapshot`, undo/redo stacks, `reset_text`, `apply_edit_with_inserted_atoms`, snapshot push limits.
- `gpui-text-input/src/widget/mod.rs`: `TextInput` retained widget fields, construction, setters, atom insertion APIs.
- `gpui-text-input/src/widget/render.rs`: layout prepaint/paint lifecycle and retention of `last_layout`.
- `gpui-text-input/src/widget/layout.rs` and `src/widget/layout/shape.rs`: current-text shaping and line layout allocation.
- `gpui-text-input/src/widget/keyboard.rs`: clipboard policies and scroll retention.
- `gpui-text-input/src/widget/ime.rs`: platform text-input and IME marked-range behavior.
