## Retained State And Lifecycle

Each `TextInput` entity owns a `TextInputState` with the current normalized UTF-8 text buffer, selection range, selection direction, optional IME marked-text range, opaque atom vector, and undo/redo stacks.

Undo and redo retain full-buffer `EditSnapshot` values. Each snapshot clones the full text string and atom vector. `TextInputOptions::DEFAULT_UNDO_LIMIT` is `128`, and both undo and redo stacks are individually limited by that count. The limit is deterministic by snapshot count, not by retained bytes.

`TextInputOptions` now bounds undo/redo history by both snapshot count and a per-stack retained-byte budget. `TextInputOptions::with_undo_limit` controls snapshot count, and `TextInputOptions::with_undo_byte_limit` controls retained snapshot bytes for each undo or redo stack. A zero count or zero byte budget disables undo recording. A snapshot that cannot fit inside the byte budget is not retained.

`TextInput::clear_edit_history` and `TextInputState::clear_edit_history` clear undo and redo stacks without changing current text. `TextInput::set_text` calls `TextInputState::reset_text`, which replaces the current buffer and clears undo and redo stacks. Beryl uses this on composer clear, history replacement, transcript-edit seeding, workspace picker reset, rename seeding, surface-notice sync, and settings-window value sync.

The widget also retains last-render layout data: `last_layout: Vec<InputLineLayout>`, `last_bounds`, `last_geometry`, scroll offsets, content height, and visible byte range. `InputLineLayout` owns GPUI `WrappedLine` values for the current rendered text. `TextInputGeometry` owns a cloned current layout so range-bound queries can be answered from the same measured snapshot. This duplicates only the current painted layout and is bounded by the current widget entity and current text, but there is no explicit byte or line cap in `gpui-text-input`.

`TextInputState::retained_counts` reports lower-bound current text, atom, undo,
and redo counts without exposing private stack internals. `TextInput::retained_counts`
adds widget layout-line, visual-line, and visible-range diagnostics from the
widget-owned layout cache. These diagnostics are intentionally app-neutral and
do not inspect GPUI shaper/glyph caches.

Layout construction allocates transient full-text and per-line structures: it clones `state.text()` to a `String`, creates logical-line ranges, atom ranges, text runs, shaped lines, selection quads, and visual-line helper vectors. These are mostly dropped after prepaint/paint except for `last_layout`.

The crate does not define global text layout caches, maps, or resource pools. It calls GPUI `window.text_system().shape_text`; any GPUI-owned glyph or shaping cache is outside this note.

## Layout Geometry

The widget builds rendered text geometry through `build_input_layout`, `shape_logical_lines`, and `WrappedLine` values in `src/widget/layout.rs`, `src/widget/layout/shape.rs`, and `src/widget/layout/geometry.rs`.

Wrapped visual-line counts are derived from each shaped `WrappedLine`'s wrap boundaries. Content height is derived from shaped logical-line block heights. Caret bounds, selection quads, generic range bounds, visible byte range, scroll offsets, and vertical reveal offsets are all computed from the same layout data used for painting.

The local patched source exposes app-neutral geometry snapshots through `TextInputGeometry`, `TextInputScrollLimits`, and `TextInputVerticalReveal`. `TextInput::measure_geometry(bounds, window)` measures proposed bounds before paint by calling the same `build_input_layout` path used for rendering. `TextInput::geometry()` returns the last painted snapshot. Cached helpers expose `bounds_for_range`, `caret_bounds`, `active_selection_endpoint_bounds`, and `scroll_limits`. `TextInputGeometry::bounds_for_range` answers generic range-bound queries from the measured snapshot.

## Clipboard, Atoms, And IME

Plain copy/cut writes only `selection.copy_text()` to the GPUI clipboard unless atom clipboard policy is `Propagate` and the selection contains atoms.

Plain paste reads clipboard text and inserts it unless rich paste policy is `Propagate` and the clipboard item has non-string entries or metadata-only content is allowed. Beryl sets `Propagate` for the composer so Beryl-owned image and rich clipboard handling runs outside the dependency.

IME composition is retained only as the current `marked_range` plus current text. `replace_and_mark_text_in_range` edits the buffer and marks the inserted range; `unmark_text` clears the range. There is no separate composition string cache beyond the normal buffer and undo snapshot path.

## Integration Gotchas

Large composer or settings-field contents can be retained multiple times: current `TextInputState.text`, Beryl or settings-window mirror state, `TextInputState` undo/redo snapshots, and `last_layout` shaped lines.

The highest-impact dependency-owned retention is the undo/redo history on multiline composer and settings text fields. Snapshot count and retained bytes are bounded by options, but the current buffer and the latest widget layout still scale with the current field contents. Host applications remain responsible for configuring field-appropriate limits and clearing edit history after accepting durable contents.

Beryl configures this at construction with `TextInputOptions::with_undo_limit` and `TextInputOptions::with_undo_byte_limit`. Beryl sets undo limit `0` for the read-only surface-notice input, sets explicit byte budgets for direct Beryl inputs, and passes a settings-field byte budget through `gpui-settings-window`.

Opaque atoms are dependency-owned only as ids, ranges, and fallback copy strings. Image bytes, durable asset ids, label scopes, and rich clipboard payloads are Beryl-owned; Beryl's `ComposerClipboardStore` has its own bounded payload count of `32`.

## Minimal Source Entrypoints

- `gpui-text-input/src/options.rs`: `TextInputOptions`, default undo limit, configuration methods.
- `gpui-text-input/src/state.rs`: `TextInputState`, `EditSnapshot`, undo/redo stacks, `reset_text`, `apply_edit_with_inserted_atoms`, snapshot push limits.
- `gpui-text-input/src/widget/mod.rs`: `TextInput` retained widget fields, construction, setters, atom insertion APIs, and widget retained-count forwarding.
- `gpui-text-input/src/widget/render.rs`: layout prepaint/paint lifecycle and retention of `last_layout`.
- `gpui-text-input/src/widget/geometry_api.rs`: public geometry snapshot types and `TextInput` measurement/cached geometry accessors.
- `gpui-text-input/src/widget/layout.rs` and `src/widget/layout/shape.rs`: current-text shaping and line layout allocation.
- `gpui-text-input/src/widget/layout/geometry.rs`: caret quad, selection quads, range bounds, visible range, and vertical reveal calculations.
- `gpui-text-input/src/widget/keyboard.rs`: clipboard policies and scroll retention.
- `gpui-text-input/src/widget/ime.rs`: platform text-input and IME marked-range behavior.
